üöÄ Features: Take Control of Obsidian Indexing

  • File Filtering & Exclusion: Define rules to specify which files/folders Obsidian should completely ignore.
  • Hide Files (Add Dot Prefix): Adds a dot (.) prefix to matched items, effectively hiding them from Obsidian's index, search, and graph.
  • Show Files (Remove Dot Prefix): Easily reverse the process and make items visible again.
  • .gitignore-Style Patterns: Use familiar and powerful patterns (e.g., node_modules/, temp/, *.log, /_site/) for **folder exclusion**.
  • True Index Control: Stops Obsidian from wasting resources processing content you don't need, leading to significant **performance improvements**.

‚öôÔ∏è How to Use: Simple Configuration

Matching Rules for File Exclusion

Supports patterns like:

  • Specific file: test.md
  • Root directory file: /config.yaml
  • Entire folder: temp/, node_modules/, .git/ (Essential for **excluding node_modules** and **excluding .git**)
  • Wildcard matching: *cache/, _build/, *.tmp

Operations: Hide or Show Files

Configure your rules in Settings ‚Üí Community plugins ‚Üí File Ignore, then click:

  • "Hide Files": Applies dot prefixes to ignore matched items.
  • "Show Files": Removes dot prefixes to un-ignore items.

üõ†Ô∏è Installation: Get Started Quickly

  1. Open Settings > Community plugins in Obsidian.
  2. Ensure Safe mode is off.
  3. Click Browse community plugins.
  4. Search for "File Ignore" (or related terms like "exclude folder", "ignore node_modules").
  5. Click Install.
  6. Once installed, click Enable the File Ignore plugin.
